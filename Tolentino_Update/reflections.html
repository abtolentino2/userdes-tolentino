Reflection 4

Userinyerface web application is not the best web application out there but it speaks the importance of having a user-friendly web application in order for the user to navigate easily and properly. Userinyerface on the other hand, is the exact opposite of a user-friendly application because it requires unusual inputs from the user leading them to stall longer from a single page to another. From my experience, upon entering Userinyerface the color scheme seems to be more contradicting from the text because green means to confirm and the green circle contains a text saying “No” so I think every user might find this unusual and wrong. Also, the direction to get in the next page is quite bothering because the underlined text is not on “here” while most of the redirecting URL does that. Entering the next page which has the form to fill-in, every text box has texts that the user should delete first before entering their desired texts. When the user gets away from the keyboard, there’s a pop-up that will indicate your time and it will not exit til the user closed it, but the thing is the close button is not on the upper-right side but instead it’s on the lower-left spelled with the copyright symbols which confuses me as a user because it is wrong again. Finishing the form will get the user into the terms and conditions but this terms has a very slow scrolling experience, but upon knowing the logic of this web-page I’ve activated the find bar and searched for the word “fast” and found an instruction that to scroll faster I should hold “Alt” button with it. The filling-in the form is the page where I’ve been stalled the most but the rest pages after it was easy for me. Just by knowing that the application is tricking me, I’ve thought of doing what is not to do properly because everything in this web application is not supposed to be user friendly. On the good side of the story this application just enlightens it user of having a uniformed and complete design that will not led its users to confusion.
